###목적 : 수집한 리뷰데이터로 인사이트를 만들기 위해, 리뷰데이터를 전처리와 필요한 데이터로 요약한다.
아래 스텝을 확인해서, 보충할건 해서 리뷰분석 프로젝트 만들거야.
프로젝트로 파이썬 코드 만들어서 실행할 수 있도록 해라.
raw데이터는 아래 경로를 참조할 것.
데이터 : data/올영리뷰데이터_utf8.csv

Step 1) “사전(딕셔너리)부터 만든다” ? 2~3시간 투자
너가 하려는 건 전부 키워드의 ‘의미’가 중요해서, 형태소/빈도만으로는 PM 인사이트가 안 나와.

1-1. 키워드 카테고리(축) 설계 (이게 포지셔닝 맵의 축이 됨)

토너에서 PM이 보는 축은 거의 정해져 있어:

(A) 효능/기능 축

진정(어성초/트러블/붉은기)

보습(속건조/수분/당김)

장벽(장벽/시카/회복)

각질/결(각질/피부결/매끈)

피지/유분(지성/번들/기름)

(B) 사용감/제형 축

물같음(물, 묽다)

쫀쫀/점성(쫀쫀, 꾸덕)

끈적(끈적, 답답)

흡수(흡수, 스며)

레이어링(여러번, 겹쳐)

(C) 사용법/역할(JTBD) 축

닦토(닦토, 닦아)

스킨팩(토너팩, 스킨팩)

레이어링 베이스(레이어, 베이스)

바디/대용량(몸, 대용량, 팍팍)

(D) 가치/선택 이유 축

가성비(가성비, 싸다, 가격)

무난/애매(무난, 그냥, 애매, 모르겠)

인생템/재구매(인생템, 몇통째, 재구매)

?? 이 축을 먼저 확정해야 “무난 점유율”, “재구매 이유”, “포지셔닝 문장”이 다 같은 언어로 뽑혀.

1-2. “무난/애매”는 따로 강제 리스트로 만든다 (가장 중요)

무난: 무난, 그냥, 평범, 무탈, 데일리, 무조건, 무색무취

애매/효능없음: 애매, 모르겠, 효과없, 잘모르, 느낌없, 물같, 밍밍

여기만 제대로 만들어도 독도 vs 경쟁제품의 차이가 숫자로 나온다.

Step 2) 리뷰를 “문장 단위로 점수화”한다 (자동화 핵심)

각 리뷰를 읽고 사람이 분류하면 시간이 무한정 든다.
그래서 리뷰 1건당 아래 변수들만 만들면 나머지 산출물은 조합해서 나온다.

2-1. 리뷰 1건에서 만들 변수(추천)

sentiment : POS/NEU/NEG

strength : WEAK/MID/STRONG

benefit_tags : [진정, 보습, 장벽, 결, 피지] 중 복수

texture_tags : [물같음, 쫀쫀, 끈적, 흡수] 중 복수

usage_tags : [닦토, 스킨팩, 레이어링, 바디] 중 복수

reason_buy : 구매이유 1개(가성비/진정/보습/대용량/기타)

reason_rebuy : 재구매이유 1개(안전성/효능/습관/가성비/기타)

switch_signal : 0/1 (갈아탐/다른제품/비교)

switch_to_brand : 브랜드명(아누아/넘버즈인/토리든/아비브/기타)

이걸 **사전 기반(키워드 룰)**로 1차 생성하고, 필요하면 GPT로 2차 보정하면 돼.

Step 3) 이제부터 산출물은 “집계”다 (네가 원하는 5개)

위 변수가 채워졌다는 가정하에, 산출물 만드는 순서는 아래가 가장 빠름.

3-A) 무난/애매 점유율 비교 (가장 먼저, 임팩트 큼)

목표: “독도는 무난/애매 언어가 압도적으로 높은가?”

제품별 neutral_keyword_rate = (무난 or 애매 태그 포함 리뷰 수) / 전체 리뷰 수

더 정확히는 분리

무난_rate

애매_rate (효능없음)

해석 템플릿

독도: 무난↑, 애매↑ → “차별성 붕괴”

토리든: 보습↑, 수분↑, 애매↓ → “효능 앵커 강함”

3-B) 키워드 포지셔닝 맵 (2D 매핑)

여기서 PM이 쓰는 가장 실전적인 맵 2개 추천.

맵1: “진정 ↔ 보습” (효능 축)

X = 진정 언급률

Y = 보습 언급률

맵2: “물같음 ↔ 쫀쫀” (사용감 축)

X = 물같음 언급률

Y = 쫀쫀/점성 언급률

제품 5개를 점으로 찍으면,
독도는 어디에 박혀 있는지 바로 보임.

3-C) 포지셔닝 문장 만들기 (자동 생성 규칙)

문장은 감으로 쓰면 설득력이 약해.
데이터로 문장 템플릿을 고정해.

템플릿

“{타겟피부/상황}이 {핵심효능}을 위해 쓰는 {사용역할} 토너”

문장에 들어갈 데이터는 이렇게 뽑아:

타겟피부/상황 = 가장 많이 언급된 skin_type/season

핵심효능 = benefit_tags Top1~2

사용역할 = usage_tags Top1

예)

독도: “민감·복합이 매일 닦토/스킨팩으로 쓰는 무자극 베이스 토너”

아비브: “트러블/붉은기 고민이 진정을 체감하려고 쓰는 어성초 토너”

토리든: “속당김이 수분 충전을 위해 레이어링하는 히알 토너”

3-D) 재구매 이유 비교 (충성도의 ‘질’ 비교)

제품별로 reason_rebuy를 집계해서 파이차트/막대로 비교해.

핵심은 ‘재구매가 같은 재구매가 아니다’

독도 재구매가 “습관/가성비” 중심이면 → 성장 정체

경쟁제품 재구매가 “효능 체감” 중심이면 → 팬덤/확장 가능

추가로 꼭 볼 것:

strength=STRONG인 리뷰에서의 재구매 이유 (진짜 팬덤)

3-E) 고객 전환 시나리오 (Switching Narrative)

리뷰에서 “갈아탐/비교/OO쓰다가”는 전환 신호야.

1) 전환 신호 탐지

switch_signal=1 조건:

“갈아탐/바꿈/다음엔/대신/비교해보니”

다른 제품명/브랜드명 등장

2) 전환 방향 집계

독도 → 아비브 / 독도 → 토리든 같은 매트릭스

그리고 전환 이유를 같이 붙여

애매(효능부족) → 진정/보습 체감 제품으로 이동

3) 시나리오 문장화

“독도(무난/효능 애매) → 아비브(진정 체감)”
“독도(물같음) → 모찌(쫀쫀 보습막)”
