🎯 1. 이 프로젝트가 뭐야?

올리브영에서 토너(스킨) 제품을 산 사람들이 쓴 리뷰 4,588개를 분석하는 프로젝트야.

학교에서 설문조사 하고 결과 분석하는 것처럼, 화장품 리뷰를 컴퓨터가 자동으로 읽고 분석하는 거야!

왜 이런 분석을 해?

  • 회사(PM)는 "우리 제품 평가가 어때?" "경쟁사보다 뭐가 좋대?"를 알고 싶어
  • 리뷰 4,588개를 사람이 다 읽으면 며칠 걸려... 😱
  • 그래서 파이썬(Python)으로 자동 분석 프로그램을 만든 거야!

분석하면 뭘 알 수 있어?

  • 📈 어떤 브랜드가 "그냥 무난해요"라는 평가를 많이 받는지
  • 📍 브랜드마다 "진정 효과", "보습 효과" 뭘 더 강조하는지
  • 🔄 사람들이 A 브랜드에서 B 브랜드로 갈아타는 패턴
  • 💝 왜 다시 사는지 (재구매 이유)

😊😐😢 5. 감성 분석 - 리뷰가 긍정인지 부정인지

리뷰가 좋은 평가인지(긍정), 나쁜 평가인지(부정), 중간인지(중립)를 판단해.

어떻게 판단해?

⭐ 별점 확인
→
📝 키워드 확인
→
✅ 최종 판정

Step 1: 별점으로 1차 판단

별점 판정
⭐⭐⭐⭐ ~ ⭐⭐⭐⭐⭐ (4-5점) POS (긍정)
⭐⭐⭐ (3점) NEU (중립)
⭐ ~ ⭐⭐ (1-2점) NEG (부정)

Step 2: 키워드로 보정

별점만 보면 부정확할 수 있어. 글 내용도 확인해!

예시:
"별 5개 줬지만 솔직히 별로예요" → 별점은 5점이지만, "별로"가 있으니 보정! 
# src/sentiment_analyzer.py

def analyze_sentiment(text, rating):
    # 1단계: 별점으로 기본 판정
    if rating >= 4:
        base = "POS"  # 4-5점은 긍정
    elif rating <= 2:
        base = "NEG"  # 1-2점은 부정
    else:
        base = "NEU"  # 3점은 중립

    # 2단계: 키워드로 보정
    긍정_단어 = ["좋아", "최고", "추천"]
    부정_단어 = ["별로", "실망", "후회"]

    # 부정 단어가 많으면 보정
    if 부정_많으면 and base == "POS":
        return "NEU"  # 긍정 → 중립으로 하향

    return base

강도(Strength) 분석

같은 긍정이어도 "그냥 좋아요" vs "인생템이에요!!!"는 다르잖아?

강도 판단 기준 예시
STRONG (강함) "인생템", "미쳤", "대박" 포함 "이거 진짜 인생템이에요!!!"
MID (보통) 특별한 표현 없음 "좋은 것 같아요"
WEAK (약함) "그냥", "무난", "보통" 포함 "그냥 무난해요"

🏷️ 6. 태그 추출 - 리뷰에서 핵심 정보 찾기

리뷰 글에서 키워드 사전에 있는 단어를 찾아서 태그를 붙여줘.

인스타그램에서 #해시태그 다는 것처럼, 리뷰에 자동으로 태그를 달아주는 거야!

예시로 이해하기

리뷰: "촉촉하고 흡수 잘되서 좋아요! 닦토로 쓰기 딱이에요. 가성비도 좋음"

자동 추출된 태그:
  • benefit_tags: [보습] ← "촉촉" 발견!
  • texture_tags: [흡수] ← "흡수" 발견!
  • usage_tags: [닦토] ← "닦토" 발견!
  • value_tags: [가성비] ← "가성비" 발견!

코드 로직

# src/tag_extractor.py

def extract_tags(text, keyword_dict):
    """
    text: 리뷰 글
    keyword_dict: 키워드 사전
    """
    found_tags = []  # 찾은 태그를 저장할 리스트

    # 사전의 각 태그를 순회
    for tag, keywords in keyword_dict.items():
        # 예: tag="보습", keywords=["수분", "촉촉", ...]

        for kw in keywords:
            if kw in text:  # 키워드가 리뷰에 있으면
                found_tags.append(tag)  # 태그 추가!
                break  # 같은 태그 중복 방지

    return found_tags

# 사용 예시
리뷰 = "촉촉하고 흡수 잘돼요"
결과 = extract_tags(리뷰, BENEFIT_KEYWORDS)
# 결과: ["보습"]
💡 핵심 로직:
if kw in text → 파이썬에서 문자열 안에 특정 단어가 있는지 확인하는 방법!
예: "촉촉" in "촉촉하고 좋아요" → True

📊 7. 5가지 분석 결과물

A 무난/애매 점유율 비교

"무난해요", "그냥 평범해요", "잘 모르겠어요" 같은 표현이 얼마나 많은지 비교해.

왜 중요해?
"무난하다"는 말이 많으면 → 제품의 특징이 뚜렷하지 않다는 뜻!
회사 입장에서는 "우리 제품만의 강점이 뭔지" 고민해야 해. 
# analysis/neutral_rate.py

# 브랜드별로 "무난" 키워드가 포함된 리뷰 비율 계산
무난_비율 = (무난_키워드_있는_리뷰_수 / 전체_리뷰_수) * 100

B 키워드 포지셔닝 맵 (2D 그래프)

브랜드들을 2차원 그래프에 점으로 찍어서 위치를 비교해.

맵 1: 효능 축
X축 = 진정 언급률 / Y축 = 보습 언급률

맵 2: 사용감 축
X축 = 물같음 언급률 / Y축 = 쫀쫀 언급률
수학 시간에 배운 좌표평면에 점 찍는 것처럼, 각 브랜드를 점으로 표시해!

C 포지셔닝 문장 자동 생성

데이터를 기반으로 브랜드별 한 줄 설명을 자동으로 만들어! 

# 템플릿
"{피부타입}이 {핵심효능}을 위해 쓰는 {사용역할} 토너"

# 결과 예시
"복합성이 보습/진정을 위해 쓰는 닦토 토너"

D 재구매 이유 분석

"재구매" 태그가 달린 리뷰에서 왜 다시 샀는지 분석해.

재구매 이유 의미
효능 "효과가 좋아서" → 팬덤 형성 가능!
안전성 "순해서, 자극없어서" → 민감피부 충성고객
습관 "계속 쓰던 거라" → 성장 정체 가능성
가성비 "싸서" → 가격 올리면 이탈 위험

"갈아탔어요", "이거 쓰다가 저걸로 바꿨어요" 같은 전환 신호를 찾아!
